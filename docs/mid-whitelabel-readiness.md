# MID 화이트라벨 — 실오픈 준비도 게이트

> 상태: **다크런치 완료 / 실오픈 미승인**
> 이 문서는 체크리스트이며, 통과 여부를 사람이 판단한다. 자동으로 실오픈하지 않는다.

## 지금 상태

- 기능·스키마·시드 전부 배포 가능 상태로 브랜치에 있다.
- MID 학원 `academies.is_active = false` — 마켓·학생 노출 없음.
- 외부 링크(인스타그램 프로필 등)는 교체하지 않았다.
- 실환불 자동 집행은 열지 않았다(제안 + 스태프 확인까지만).

## 실오픈 전 남은 것

- [ ] **운영자 매뉴얼** — 인포데스크가 볼 수 있는 절차서(입금확인·현장결제·휴강·태깅·재처리)
- [ ] **환불 실집행 개방 여부 결정** — 현재는 제안까지만. 자동 집행을 열지, 계속 수동 확인으로 둘지
- [ ] **기존 12개 테이블 RLS 부채** — 이번 범위 밖. 실고객 데이터 유입 전 처리 여부 판단
- [ ] **재처리 큐 0건 확인** — 오픈 시점에 미처리 잔여가 없어야 한다
- [ ] **내부 파일럿** — 제한된 실제 계정으로 예약·취소·QR·입금확인 왕복
- [ ] **대표 단발 승인** → 그 후에만 `is_active` 전환 + 외부 링크 교체

## 알려진 미해결 사항

1. **만료일 계산 두 계보** — 기존 수강권은 `start + valid_days`, 신규 규칙은 포함일수. 기존 의미는 보존됐으나 규칙 통일은 제품 결정이 필요하다.
2. **환불 프리셋은 옵트인** — 기존 상품에 자동 적용되지 않는다. 이관은 별도 결정.
3. **카드 결제 실물 왕복 미검증** — PG 응답 이후 코드는 전부 검증됐으나 실제 결제창 왕복은 테스트에서 스텁으로 대체됐다. 파일럿에서 실물 1건 확인 권장.
4. **일부 GET 라우트 캐시 처리 미완** — 운영자 화면 밖 경로. 별도 작업으로 분리됨.
5. **월등록·3개월 상품의 커버리지** — 수업군 기준으로는 정규 전체를 커버하고, "고른 반에서만" 제약은 런타임(`fixed_class_id`)에서 걸린다. 동작은 맞으나 커버리지만 보면 넓어 보인다.
