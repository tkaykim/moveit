1.	수강권의 표기
A.	기간제 수강권 (기존의 ‘정규 클래스 수강권’ 워딩을 ‘기간제 수강권’으로 표기)
특정 기간동안 사용할 수 있는 수강권이며, 사용시 횟수가 차감되거나 하는 개념은 아니고, 시작일로부터 종료일까지 사용할 수 있는 수강권임.
사용자가 구매 또는 관리자로부터 얻어 수강권을 획득한 경우, 특정 시작일을 지정할 수 있음. 또는 수업신청시 사용된 첫 수업 일자 기준으로 유효기간 카운팅 시작되어야함. (시작일은 해당 수강권으로 들을 수 있는 연결된 수업이 있는 날만을 선택할 수 있음)
해당 수강권으로는 어떤 수업들을 들을 수 있는지 체크할 수 있어야함.
B.	쿠폰제(횟수제) 수강권 (기존 팝업 수강권 워딩을 ‘쿠폰제(횟수제) 수강권’으로 표기)
이름을 정하고, 각 수강권에 횟수별 가격과 유효기간을 정할 수 있어야 함.(현재 구현된 상태.)
“취미반 쿠폰-1,3,5회 권”, “전문반 쿠폰-1,10,30회 권” 등으로 쿠폰이름 설정 및 쿠폰별 옵션 (횟수, 유효기간)을 설정할 수 있어야 한다는 뜻.
마찬가지로, 해당 수강권으로는 어떤 수업들을 들을 수 있는지 체크할 수 있어야 함.
C.	워크샵 수강권 (워딩을 ‘워크샵(특강) 수강권 으로 표기)
이름을 정하고, 해당 수강권으로 들을 수 있는 수업을 체크할 수 있어야 함. 유효기간을 설정할 수 있음.

2.	회원 관리
A.	회원 목록을 검색하거나 필터를 통해 걸러볼 수 있어야 함.
B.	현재 특정 회원이 어떤 수강권들을 몇 개 가지고 있고, 유효기간, 잔여 횟수 등을 어떻게 되는지를 간단히 볼수 있어야 하고, 클릭시 상세보기로 자세히도 볼 수 있어야함. 자세히 보기에는 인적사항과 등록일자, 보유중인 수강권의 종류와 유효기간등을 효율적으로 볼 수 있어야 함.

3.	유효기간 연장 신청 또는 pause 관리. 
A.	사용자가 my페이지 (/my) 어딘가에서 구매한 기간제 수강권이나 쿠폰 수강권에 대해 유효기간을 연장 신청하거나 유효기간 일시정지 (비슷한 개념임.) 를 신청하고
B.	관리자가 그걸 학원 관리자 페이지에서 “수강권 연장/일시정지 관리”탭에서 관리 (승인,거절 등)할 수 있어야 함. 승인 또는 거절한 경우 사용자가 알 수 있어야 함. 관리자가 거절 시에는 거절 사유를 입력해야함. 
C.	유효기간 연장은 유효기간 연장을 신청할 수강권을 선택하고, absent 할 기간 (예를 들면 휴가 기간, 시험기간 등을 설정해 시작일부터 종료일을 선택하면 해당 일자만큼이 늘어나야함. 연장신청이 최대로 설정 가능한 기간(일수)는 학원마다 정책이 다르므로, 관리자가 설정할 수 있어야함.
D.	관리자가 특정 회원에 대해 임의로 유효기간 연장이나 일시정지를 할 수도 있어야 함.

4.	신청인원 관리 (출석/신청 관리 로 워딩 변경)
A.	날짜를 선택하면 해당 날짜에 수업을 들을 에정인 사람들이 보여야함. 수업을 선택할 때에 현재는 같은 수업임에도 날짜별로 드롭다운이 다 나오고 있는데, 날짜를 선택하고 나면 수업명은 class를 기준으로 하나만 나오면 됨.
B.	수기 추가(관리자 권한)의 경우, 수강생을 검색해서 찾아 넣는 방식이 아니라, 이름과 연락처를 직접 기입하고 사유를 기입하여 추가할 수 있어야 합니다.
C.	현재 ‘게스트’라는 워딩은 ‘비회원’으로 변경하고, 관리자가 직접 입력한 수기 추가자를 ‘게스트’로 표기하세요.

5.	수강권 판매 시스템
A.	수강권 선택시 기간제, 쿠폰제, 워크샵이 구분되어 보여지도록 구현되어있는지 확인하고, 아니라면 그렇게 보이도록 개선하세요

6.	관리자가 수강자에 대해 취소해주는 경우, 기간제는 상관 없지만, 쿠폰제의 경우 취소했을 때 보유 수량이 다시 구매자(회원)에게 돌아갈지를 선택할 수 있어야 합니다.

7.	기간제 수강권의 수업신청 로직.
A.	현재 쿠폰제(횟수제, 기존표현으로는 팝업) 클래스의 경우 모든 수업이 그때 그때 신청되는 개념이지만, 기간제 수강권의 경우 특정 수업에 대해 시작일을 설정하면, 시작일~종료일 사이의 모든 해당 수업에 이 수강생이 자동으로 수강신청이 되어있어야 합니다. 
B.	수강권 연장, 일시정지가 승인된 경우 그에 맞게 수강 인원에서 빠지고 늘어난 기간에 대해 그 수업에는 추가로 명단이 들어가야 하는 개념입니다.

8.	수강권 구매 링크. 생성된 수강권에 대해 수강권 구매 링크를 생성할 수 있도록 하고, 사용자는 그 링크를 타고 들어왔을 때에, 어떤 학원의 어떤 수업인지 확인하고 결제를 할 수 있어야 하며, 이 때에도 기간제 수강권의 경우 시작일자를 정할 수 있어야 함. (해당 수강권이 연결된 수업이 있는 날만을 선택할 수 있음)

9.	수강 신청(결제)페이지 (book) 페이지의 개선사항.
A.	현장결제를 클릭한 순간 정말 현장결제로 진행하시겠어요? ‘현장 결제의 경우 결제 선착순 마감으로 수업에 참석하실 수 없을 수 있습니다’ 를 팝업하여 확인 받고 진행.
B.	비회원 결제가 쉽도록 하세요. 이름, 연락처 (또는 이메일) 등의 간단한 정보만 기입했을 때에도 바로 결제할 수 있어야 하고, 관리자는 신청자 명단에서 비회원 신청도 확인할 수 있어야 합니다.

10.	상담 신청 기능.
A.	특정 학원 홈에서 상담 신청하기 기능을 구현하고 싶으며, 관리자의 상담관리 탭과 연동해야합니다.
B.	상담 신청 양식은 이름, 연락처, 카테고리, 상세 내용, 방문 예정 일시(날짜와 시각(시각은 30분 단위))을 구현하세요
C.	관리자의 상담관리 탭에서는 상담 카테고리 추가/삭제를 할 수 있어야 하며 카테고리 이름, 소요시간(분)을 입력할 수 있어야 합니다. (ex. 입시반 30분, 오디션반 30분, 전문반 30분, 일반 상담 10분을 프리셋으로 만들어 두도록 하세요)
D.	관리자의 상담관리 탭에서는 상담 가능 시각을 관리자가 직접 설정할 수 있어야 합니다. 전화 상담 가능 시간, 방문 상담 가능 시간
